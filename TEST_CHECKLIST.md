# 将棋解析アプリ 試作テストチェックリスト

## 1. 事前準備
- [ ] `npm run dev -- --host` で起動する
- [ ] PCブラウザで画面が開く
- [ ] 同じURLを Pixel 8a から開ける

## 2. 基本動作
- [ ] サンプル棋譜で盤面が表示される
- [ ] 手数スライダーで局面が変わる
- [ ] 「戻る」「進む」「初期局面」が動く
- [ ] 棋譜一覧をタップして該当手へ飛べる
- [ ] 候補手と評価値が表示される

## 3. 棋譜入力
- [ ] テキスト欄へ簡易棋譜を貼って再生できる
- [ ] KIFファイルを読み込める
- [ ] CSAファイルを読み込める
- [ ] 読み込み後に source 表示が変わる
- [ ] 読み込みエラー時に画面が固まらない

## 4. ローカル保存
- [ ] 棋譜を編集してページ再読込しても残る
- [ ] 現在手数が再読込後も残る
- [ ] 読み込んだファイル名が残る
- [ ] 「保存内容をリセット」で初期状態に戻る

## 5. Android PWA テスト (Pixel 8a)
- [ ] Chromeでページを開く
- [ ] 「ホーム画面に追加する」導線を確認できる
- [ ] 追加後、ホーム画面アイコンから起動できる
- [ ] ブラウザUIが薄い or standalone風に見える
- [ ] 再起動しても保存データが残る

## 6. オフライン寄り確認
- [ ] 一度開いた後に通信を切って再度開ける
- [ ] 少なくとも基本UIが表示される
- [ ] 保存済み棋譜が読める

## 7. スマホUI
- [ ] 盤面が横にはみ出しすぎない
- [ ] ボタンが押しやすい
- [ ] テキスト欄が編集できる
- [ ] スクロールが極端に壊れていない
- [ ] safe areaで上端下端が詰まりすぎない

## 8. iPhone確認ポイント（後日でOK）
- [ ] Safariで開いてレイアウト崩れがない
- [ ] 共有 → ホーム画面に追加が案内通りできる
- [ ] ホーム画面起動で大きな崩れがない
- [ ] 保存データが保持される

## メモ
- まずは Android 実機で 1〜7 を回すのがおすすめ
- 問題が出たら、どの操作で何が起きたかを1行でメモすると修正が速い
