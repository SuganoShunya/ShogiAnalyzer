# 実機テスト手順

## 前提
- PC と Android / iPhone が同じ Wi-Fi にいる
- このアプリの workspace で起動する

## 1. 開発サーバー起動
```bash
npm run dev
```

この試作は `--host 0.0.0.0` で起動するので、LAN から開ける。

## 2. スマホから開く
このPCのLAN IPが `192.168.11.22` の場合:

- <http://192.168.11.22:5173>

iPhone なら Safari, Android なら Chrome 推奨。

## 3. 基本動作テスト
- 棋譜入力
- KIF/CSA読込
- 棋譜再生
- 盤面タップ試し指し
- 成る / 成らない
- 持ち駒から打つ
- 候補手タップ
- PVタップ
- PV/試し指し解除
- ホーム画面追加

## 4. USIテスト
1. PCにUSIエンジンを置く
2. アプリの `USI設定` にパスを入れる
3. `USI接続テスト` を押す
4. 接続OKになったら解析モードが `PCエンジン` になるか確認

## 5. つまずきやすい点
- Windows Firewall で 5173 が塞がれる
- PC とスマホが別Wi-Fi / ゲストWi-Fi
- エンジンパスが間違っている
- iPhoneでPWA追加前はSafari経由の動作差がある

## 6. 見るポイント
- 接続状態が `接続OK` か
- `解析モード` が `PCエンジン` か `軽量フォールバック` か
- PVプレビュー中でも操作が崩れないか
